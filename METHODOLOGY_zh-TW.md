# RiskLens 信用評級方法論

本文件概述了 RiskLens 產生內部信用評級和風險評分所使用的量化框架。該方法論將經典的多元破產預測模型（**Altman Z-Score 模型**）與現代持續的貸後契約監測相結合。

---

## 1. 核心評分架構：Altman Z-Score

RiskLens 高度依賴 **Altman Z-Score** 作為公眾公司的核心演算法引擎。Z-Score 是一種多元公式，它將企業財務健康的五個不同維度綜合成為一個單一的預測分數。

**1.0 範圍說明：** 目前 API 的評分模型僅使用 Altman Z-Score。`src/credit_risk_assessment.py` 為歷史/實驗性框架，未接入 FastAPI 服務。

### Z-Score 計算公式
`Z = 1.2(X1) + 1.4(X2) + 3.3(X3) + 0.6(X4) + 1.0(X5)`

其中：
- **X1 (營運資金 / 總資產)**：評估短期流動性和財務緩衝能力。
- **X2 (保留盈餘 / 總資產)**：衡量隨時間推移的累積獲利能力，對新成立或長期虧損的企業予以低分懲罰。
- **X3 (EBIT / 總資產)**：隔離了稅收和槓桿效應，是對營運效率和資產生產率的純粹衡量指標。
- **X4 (權益市值 / 總負債)**：引入市場資本化，以評估市場對企業償付能力相對於其帳面債務的看法。
- **X5 (營收 / 總資產)**：衡量商業模式的整體資產週轉率和資本密集度。

### Z-Score 風險區間劃分
未經調整的 Z-Score 被轉換為三個不同的風險層級：
1. **安全區 (Safe, Z ≥ 2.99)**：意味著強健的財務狀況；2 年內破產的可能性在統計上可以忽略不計。
2. **灰色區 (Grey, 1.81 ≤ Z < 2.99)**：表明處於中度違約風險之中；需要加強監測並可能需要收緊信貸額度。
3. **違約危機區 (Distress, Z < 1.81)**：在 24 個月內發生違約或大規模債務重組的可能性極高。

## 2. 隱含機構評級映射表

為了將原始的 Z-Score 轉換為與標準信貸政策相容的機構性話語體系，RiskLens 將上述風險區間映射到了標準普爾 (S&P) 等效的隱含評級上。

| Z-Score 範圍 | 風險區間 | 隱含評級 | 風險類別 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| ≥ 4.50 | Safe | **AAA / AA** | 極優級 (Prime) | 履行財務承諾的能力極強。 |
| 2.99 - 4.49 | Safe | **A** | 中上級 | 履約能力強，但容易受到不利經濟環境改變的影響。 |

## 3. 貸後契約監測體系

Z-Score 提供的是某一時點下的橫截面違約機率，而財務契約則作為持續的「早期預警系統」。RiskLens 會主動監測這些關鍵指標，以便在技術性違約成為既定事實前觸發警報。

### 標準監測的財務契約：
1. **槓桿測試 (Leverage Test)**：`總負債 / EBITDA < [閾值]` — 防止借款人透過內生性成長或過度併購進而導致負債比率過高。
2. **覆蓋率測試 (Coverage Test)**：`EBITDA / 利息費用 > [閾值]` — 確保企業保留足夠的支持營運所需的現金，用於支付償債成本。
3. **流動性測試 (Liquidity Test)**：`流動資產 / 流動負債 > [閾值]` — 確保對即將到期的短期負債有足夠的資產覆蓋範圍。

*註：在貸後監測中，RiskLens 對 `null`（空值）或缺失值的處理極為保守，系統不會預設通過這些契約測試，而是會自動假設為觸發「違約 (BREACH)」，從而強制要求信貸審批人員進行人工覆核。*

## 4. 全球財務資料本體規範化

RiskLens 在不同國家和監管準則的主體之間執行極其嚴格的資料清理與對齊，以此為評分引擎提供準確的彈藥：
- **美國 / 全球模型 (yFinance)**：將標準的 SEC XBRL 標籤統整至 RiskLens 的第一主變數中。
- **中國 A 股及港股 (AKShare)**：動態地將軍團級別的中國大陸 GAAP 報表科目分類（例如，*「長期借款」* 對應 Long Term Debt，*「營業總收入」* 對應 Total Revenue）精準映射到相同的底層數學資料結構中。這保證了即便是跨越國界的各類評級，在 RiskLens 面前依然能維持絕對的公允和可比性。
