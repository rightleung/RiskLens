# RiskLens 信用格付けメソドロジー

Language: [EN](./METHODOLOGY.md) | [简中](./METHODOLOGY_zh-CN.md) | [繁中](./METHODOLOGY_zh-TW.md) | [日本語](./METHODOLOGY_ja.md)

本書は、RiskLens が内部信用格付けとリスクスコアを算出するための定量フレームワークを説明します。古典的な多変量倒産予測（**Altman Z-Score**）と、継続的な貸出後コベナンツ監視を統合しています。

---

## 1. 主要スコアリング構造：Altman Z-Score

RiskLens は上場企業評価の中核エンジンとして **Altman Z-Score** を採用しています。Z-Score は企業の財務健全性を 5 つの次元で統合し、単一の予測スコアに変換する多変量式です。

**1.0 スコープ注記：** 本番 API パイプラインは Altman Z-Score のみを使用します。`src/credit_risk_assessment.py` のレガシー評価ロジックは FastAPI サービスでは呼び出されません。

### Z-Score 公式
`Z = 1.2(X1) + 1.4(X2) + 3.3(X3) + 0.6(X4) + 1.0(X5)`

内訳：
- **X1（運転資本 / 総資産）**：短期流動性とバッファ余力を評価。
- **X2（利益剰余金 / 総資産）**：累積収益力を評価。新設企業や慢性的赤字企業には厳しく作用。
- **X3（EBIT / 総資産）**：税・レバレッジの影響を除いた営業効率と資産生産性を評価。
- **X4（株式時価総額 / 総負債）**：市場評価を取り込み、簿価負債に対するソルベンシーを評価。
- **X5（売上高 / 総資産）**：資産回転効率と資本集約度を評価。

### Z-Score ゾーン分類
生の Z-Score は以下 3 区分に変換されます。
1. **Safe Zone（Z ≥ 2.99）**：財務は堅牢。2 年以内の倒産確率は統計上低い。
2. **Grey Zone（1.81 ≤ Z < 2.99）**：中程度のストレスリスク。監視強化や与信条件の引き締めを推奨。
3. **Distress Zone（Z < 1.81）**：24 か月以内にデフォルトまたは大幅な再編が発生する可能性が高い。

## 2. インプライド格付けマッピング

生の Z-Score を実務上の与信言語に変換するため、RiskLens は各ゾーンを S&P 相当のインプライド格付けへマッピングします。

| Z-Score 範囲 | ゾーン | インプライド格付け | リスク区分 | 説明 |
| :--- | :--- | :--- | :--- | :--- |
| ≥ 4.50 | Safe | **AAA / AA** | Prime | 債務履行能力が極めて高い。 |
| 2.99 - 4.49 | Safe | **A** | Upper Medium | 履行能力は高いが逆風時に影響を受けうる。 |
| 2.50 - 2.98 | Grey | **BBB** | Lower Medium | 履行能力は十分だが保護余力は中程度。 |
| 1.81 - 2.49 | Grey | **BB** | Speculative | 直近耐性はあるが不確実性が高い。 |
| 1.20 - 1.80 | Distress | **B** | Highly Speculative | 事業・財務逆風に対して脆弱。 |
| < 1.20 | Distress | **CCC / D** | Substantial Risk | 現在脆弱で、外部環境に強く依存。 |

## 3. 貸出後コベナンツ監視

Z-Score は時点のデフォルト確率を示します。一方、コベナンツは継続的な「早期警戒システム」として機能します。RiskLens は主要指標を継続監視し、テクニカルデフォルト前にアラートを出します。

### 標準監視コベナンツ
1. **レバレッジテスト**：`Debt / EBITDA < [Threshold]` —— 借り手の過剰レバレッジ化を抑制。
2. **カバレッジテスト**：`EBITDA / Interest Expense > [Threshold]` —— 債務返済コストを賄う営業キャッシュの確保。
3. **流動性テスト**：`Current Assets / Current Liabilities > [Threshold]` —— 短期資産で直近期負債をカバーできるかを確認。

*注記：RiskLens は貸出後監視において `null` や欠損値を保守的に扱い、暗黙 PASS にせず BREACH（要手動審査）として扱います。*

## 4. 財務データ正規化

RiskLens はスコア計算精度を担保するため、地域横断で財務データを厳密に正規化します。
- **米国/グローバル（yFinance）**：標準 SEC XBRL タグを RiskLens の主変数へ統一。
- **中国 A 株（AKShare）**：中国会計科目（例：*长期借款*、*营业总收入*）を同一の数理構造へ動的マッピングし、市場横断の比較可能性を確保。
