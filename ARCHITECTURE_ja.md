# RiskLens アーキテクチャの概要

RiskLens は、機関投資家レベルの信用リスク分析を処理するために設計された、最新の疎結合アーキテクチャに基づいて構築されています。システムは、RESTful JSON API を介して接続された、高性能な Python 分析バックエンドとレスポンシブな React フロントエンドに分割されています。

## システムアーキテクチャ

### 1. バックエンドゲートウェイ (FastAPI)
バックエンドは **FastAPI** で構築されており、高スループットの非同期実行、自動 OpenAPI ドキュメント作成、および厳密なデータ検証を提供します。
- **`api.py` (オーケストレーター)**: `POST /api/v1/assess` エンドポイントを公開します。ティッカーシンボルの配列を受け取り、データのフェッチと翻訳プロセスを調整します。
- **並行実行**: 応答遅延を劇的に削減するために、多言語セマンティックマッチング (Google Translate) は、Python の `concurrent.futures.ThreadPoolExecutor` を使用して動的に実行され、英語、簡体字中国語、繁体字中国語、および日本語の名前を並行して取得できます。

### 2. データ取得レイヤー (`data_fetcher.py`)
このレイヤーは、外部 API からの財務データセットの物理的な取得を処理します。堅牢なフォールバック手法を実装しています。
- **AKShare (中国 A 株 および 香港株)**: 中国本土株式 (例: `600673.SS`) の主要パイプラインです。本土の API (`stock_financial_report_sina`, `stock_individual_info_em`) を直接照会し、複雑な中国の会計基準 (GAAP) の分類体系を処理し、不規則な報告期間 (第1四半期、上半期、第3四半期) を年次化された標準フォーマットに変換します。
- **yFinance (グローバル/米国)**: 米国/欧州市場の主要エンジンとして機能し、オンショア API が接続を制限した場合の安全なフォールバックとして機能します。

### 3. 分析および監視エンジン
- **`ratio_analyzer.py`**: 取り込まれた生データを 30 以上の高度な信用リスク比率 (流動比率、FCF/有利子負債、営業利益率など) に処理する純粋な数学エンジンです。
- **Altman Z-Score モデル**: 最終的な比率は Z-Score エンジンに入力され、最終的な確率的評価 (Safe, Grey, Distress) を生成します。
- **`covenant_monitor.py`**: 厳格な事前定義された制約 (例: 有利子負債/EBITDA < 3.5倍) に対して、計算された比率を評価する専用モジュールです。重要なのは、システムが保守的に失敗することです。必要なメトリックが欠落している (`null`) 場合、デフォルトで合格するのではなく、テクニカル違反の警告をトリガーします。

### 4. フロントエンドクライアント (React 19 + Vite)
- **状態管理**: 関数型 React Hooks を利用するシングルページアプリケーション (SPA) として構築されています。ダークモード (`useTheme`)、アクティブな企業データセット、およびレスポンシブな財務モーダルの中央制御ロジックが含まれています。
- **UI/UX**: クラス (`glass-panel`, `glass-header`) を介してグラスモーフィズム ユーティリティを利用し、プレミアムで機関投資家らしい美観を確立するために、`Tailwind CSS` で広範囲にスタイル設定されています。
- **ローカリゼーション (i18n)**: 650 以上のローカライズされた用語をマッピングする `translations.ts` によって決定されます。UI は、ユーザーのアクティブな言語選択 (`zh-CN`, `zh-TW`, `en`, `ja`) に基づいて、全体的なレイアウトから特定の財務の行項目に至るまで、インターフェイス全体を動的に再レンダリングします。

## アーキテクチャの重要な決定

| 決定事項 | 根拠 |
|----------|-----------|
| **非同期 FastAPI + Pydantic** | 受信リクエストに対して高スループットの並行性と自動型強制を提供し、分析エンジンが実行される前にデータ構造を保証します。 |
| **バイリンガルな分類体系のマッピング** | ピア比較を有効にするために、AKShare の中国 GAAP 用語を標準化されたグローバル構造 (IFRS/US GAAP) に正確にマッピングするために不可欠です。 |
| **評価とコベナンツの分離** | 融資後の監視 (コベナンツ) はデフォルト確率スコアリング (Z-Score) とは独立して動作し、借り手ごとに異なるしきい値を許可します。 |
| **リアルタイムなセマンティック翻訳** | `ThreadPoolExecutor` を使用することで、企業の名前が JP/ZH 市場で正確に翻訳され、大規模なデータ取得ペイロードを遅らせることはありません。 |
